#cocos2dx内存管理

管理过程:

  1. cocos2dx采用引用计数的方式进行内存管理的，当一个对象的引用计数为0的时候就会被引擎自动delete掉，在cocos2dx中，大部分类都继承CCObject类

m_uReference就是引用计数，在对象构造的时候，m_uReference被置为1

然后每次对对象进行retain操作，reference + 1;

每次对对象进行release操作，reference - 1,如果reference在这次release之后变为0，那么delete掉该对象

getCurReleasePool()返回一个CCAutoreleasePool对象指针，也就是一个自动释放池

m_pManagedObjectArray数组是接受自动释放的对象，也就是说进行autorelease的对象最终被放到该数组里面,并在addObject(pObject)的时候对这个对象进行一次retain()操作，使得对象的引用+1，那么对象在加入该数组的时候引用应该变为2，所以在addObject函数中还需要release()一次使得这次add造成的引用取消，这样一来对象从创建开始引用变为1，到现在被放进自动释放池中后引用仍然是1只是管理状态被置为true

#总结

  1. cocos2dx采用引用计数的方式进行内存管理，谁需要引用这个对象，就对其retain()一次，同时在不需要它的时候就要对其进行release()操作，在addChild()操作时，会对child进行retain()表示对其引用,而在removeChild()的时候会对其进行release()操作.
  2. 对象的autorelease()操作的效果时对象在当前这一帧被new出来后，在下一帧之前没有被执行retain()，那么这个对象在下一帧就会被引擎delete。在手动retain()后放如autorelease池，这样引擎只能将对象的引用减1，而不会delete，从而造成内存泄漏；
  3. 自动管理就是在这一帧结束的时候将new出来对象时的那个引用计数减掉，而让引擎中持有对象引用的其他类去管理这个对象，当持有者析构的时候就会删除该对象；
  4. 加入对象池的sprite的所有权属于内存池，返回的指针其实是没有所有权的。正常情况下，在创建出一个对象后，接下来的操作往往是把这个sprite对象添加到上层父节点中，此时由内存池和这个父节点共同拥有这个对象，而对象不再需要使用的时候直接通过removeChild将对象从父节点中移除，就回到对象仅属于内存池的情况了。而cocos2dx中实际每帧都会清理内存池，也就是说每一帧仅仅属于内存池的对象都会被释放。这样开发者就可以放心地将对象放在autoreleasePool中，上层的父节点通过addChild有了对象所有权以后能够保证该对象不会被删除。
