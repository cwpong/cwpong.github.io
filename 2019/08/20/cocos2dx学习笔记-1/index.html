Cocos2dx总体框架

ApplicationProtocol :

应用程序协议，是一个接口

Application :

    实现了ApplicationProtocol接口规则的调用，负责提供应用程序初始化接口（通过AppDelegate重载虚函数），实现消息循环。 
主要成员变量：
    static Application *sm_pSharedApplication，这个变量保存了唯一的Application实例的地址。
主要成员函数：
    virtual int run();循环处理应用程序消息，win32的环境下，run函数同时处理帧更新消息和windows消息。该函数中回调applicationDidFinishLaunching函数，即实现了应用程序的初始化（这个函数时纯虚函数，要求用户实现，一般是在AppDelegate中实现）

Dirctor :

    导演类，在Applicttion::run()中调用到了Dirctor::getInstance()->mainLoop函数；这个类使用单例模式实现；

导演类的作用:
    1. 在开始游戏和结束游戏时负责游戏的初始化和销毁工作。
    2. 管理游戏场景以及场景的切换。
    3. 设置、获取系统消息。比如调整OpenGL相关的设置，获取屏幕的大小等

主要成员函数：
    //获取OpenGL View的大小 单位为点
    Size getWinSize(void);

    //获取OpenGL View显示区域大小 单位为点
    //如果没调用EGLView::setDesignResolutionSize()，则值和getWinSize()相等
    Size getVisibleSize();

    //运行场景
    void runWithScene(Scene *pScene);
    void pushScene(Scene *pScene);
    void popScene(void);

    //绘制场景
    void drawScene(void);

    //初始化函数初始化了Scheduler（调度器）, ActionManager(动作管理器), TouchDispatcher(触屏管理器), KeypadDispatcher(派发器), 并申请内存自动清理管理器PoolManager()->push();
    init();

总结:

cocos2dx框架创建AppDelegate单例对象

cocos2dx框架创建EGLView单例对象并注册win窗口类和win窗口过程回调函数

cocos2dx框架在EGLView类的win窗口过程回调函数中创建了Director对象并初始化了Director对象中的管理功能（调度器、动作管理器、内存释放池等）

cocos2dx框架的AppDelegate的run函数调用ApplicationDidFinishLauching()入口， 完成界面元素的创建

cocos2dx框架的AppDelegate的run函数按照默认帧率循环调用Director::getInstance()->mainLoop()来不断显示win窗口
